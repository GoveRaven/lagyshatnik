jsDoc - способ документривания кода. Представляет собой многострочный комментарий с дополнительной звёздочкой в начале.

Если нужно записать объект, то используется следующая запись {{...}}

Теги

  • @fileoverview - базовый тег для описания файлов. Под этим тегом указывается общая информация о файле

  • @author - используется для указания автора кода

  • @const {...} или @constant {...} - обозначает постоянные значения. В фигурных скобках указывается тип даннных

  • @type {...} varName - указывает тип переменной или константы (имя переменной можно опустить), записання после фигурных скобок.

    Типами могут быть: number, string, boolen, Element. Тип объекта, который может стать null обознаается дополительно вопросительным знаком {?...}. Чтобы указать несколько возможных типов используются ||, например: {...||...}

    Описывая массивы, необходимо указать тип Array и после точки в треугольных скобках указать тип данных, находящихся в массиве: @type{Array.<number>}

    Обекты описываются также, как и массивы, но в треугольных скобках указываютя 2 типа: ключи и значения. Если значения в объекте могут принимать несколько типов, то они перечисляются через знак |: @type {Object.<string, number|string>}

  • @typedef - описывает новый тип данных. После того как написали этот тег, можно дальш пользоваться названием этой переменной как типом. Например:

    @typedef {suit: string, value: number}
    let Card;
    @param {Card} myCard
    function beatACard(myCard) {...}

  • @(constructorsName) - объявление конструктора считается объявлением нового типа данных. После этого можно использовать имя этого конструктора как тип в теге @type

  • @enum {...} - перечисляемый тип - список констант. Удобен для использования когда существует ограниченный список значений. Внутри фигурных скобок описывается какие типа могу принимать значения внутри объекта.

  • @param {...} paramName - содержит имя, тип и описание параметра функции.

    Необязательные параметры могут быт не переданы. В этом случае они будут равны undefined. В этих случаях после типа данных ставится знак равенства после типа данных или имя необязательного параметра пишется в квадратных скобках [paramName]. Можно задать необязательным параметрам значение по умолчанию: [paramName = defaultValue]

  • @return {...} - документитрует значение, возвращаемое функцией.
